### Глядченко Glossary of terms — словник термінів

- **Virtual machine (VM)** — a software-based computer inside a real one.  

- **Virtualization** — running multiple systems on one physical machine.  

- **Hypervisor** — software that creates and manages virtual machines.  

- **Host operating system (Host OS)** — the main operating system of a computer.  

- **Guest operating system (Guest OS)** — an operating system running inside a virtual machine.  

- **Kernel** — the core part of an operating system.  

- **Linux kernel** — the core of the Linux operating system.  

- **Distribution (distro)** — a complete Linux system with software and tools.  

- **Open source** — software with freely available and modifiable source code.  

- **Command Line Interface (CLI)** — a text-based way to control a computer.  

- **Graphical User Interface (GUI)** — a visual interface with windows and icons.  

- **Operating system (OS)** — software that manages hardware and programs.  

- **Java Virtual Machine (JVM)** — a virtual machine that runs Java programs.

### Козуб Охарактеризуйте поняття «гіпервізор». Які бувають їх типи?
Гіпервізор - це програмне забезпечення або мікрокод, який дозволяє декільком ОС одночасно працювати на одному комп'ютері, розділяючи його ресурси.
Типи гіпервізорів:

 - Автономні / Bare Metal: Запускаються безпосередньо на апаратному забезпеченні. Вони керують ресурсами комп'ютера та гостьовими ОС без посередництва основної ОС. Приклади: Xen, KVM, VMware ESXi, Hyper-V
  
 - Залежні / Hosted: Запускаються як звичайні програми всередині вже встановленої (хостової) ОС. Вони використовують ресурси через хостову ОС. Приклади: Oracle VM VirtualBox, VMware Workstation
   
### (1) Основні компоненти та можливості гіпервізора, Козуб 11-й варіант(VirtualBox):

 - Кросплатформеність: Може бути встановлений на Windows, Linux, macOS та Solaris
   
 - Підтримка багатьох ОС: Дозволяє запускати Windows (від 3.1 до 11), Linux, BSD, Solaris, OS/2 тощо
   
 - Snapshots (Знімки стану): Можливість зберегти стан віртуальної машини та миттєво повернутися до нього у разі помилки
   
 - Guest Additions (Доповнення гостьової ОС): Спеціальні пакети драйверів, що забезпечують спільний буфер обміну між хостом і гостем, спільні папки, покращену відеопродуктивність та інтеграцію вказівника миші
   
 - Гнучке налаштування мережі: Підтримка режимів NAT (вихід в інтернет), Bridged (ВМ як окремий пристрій у локальній мережі), Host-only (мережа тільки між хостом і ВМ)
   
 - Віртуалізація USB-пристроїв: Можливість підключати USB-пристрої безпосередньо до гостьової ОС.
   
 - Портативність: Віртуальні машини зберігаються у вигляді файлів, які легко переносити між різними комп’ютерами

### (2) Основні компоненти та можливості гіпервізора, Глядченко 4-й варіант(KVM):

 - Модуль ядра KVM: Забезпечує апаратну віртуалізацію в Linux (Intel VT-x / AMD-V)

 - QEMU: Емуляція пристроїв та запуск віртуальних машин у користувацькому просторі

 - Libvirt: API та інструменти для керування віртуальними машинами

 - Інструменти керування: virsh (CLI), virt-manager (GUI), virt-install (створення VM)

 - VirtIO драйвери: Паравіртуалізовані пристрої для підвищення продуктивності (диски, мережа)

 - Мережева віртуалізація: Bridge, NAT, VLAN, віртуальні мережеві інтерфейси

 - Підтримка багатьох ОС: Linux, Windows, BSD та інші

 - Snapshot-и: Збереження стану VM для відновлення у разі помилки

 - Live Migration: Міграція віртуальних машин між фізичними хостами без зупинки

 - Ізоляція та безпека: Кожна VM працює як окремий процес; підтримка SELinux/AppArmor

 - Інтеграція з хмарними платформами: OpenStack, KubeVirt

## Глядченко Відповіді на питання

### 1. Етапи розгортання ОС у VirtualBox
1. Встановити VirtualBox.  
2. Створити нову віртуальну машину.  
3. Обрати тип і версію ОС.  
4. Виділити оперативну пам’ять (RAM).  
5. Створити віртуальний жорсткий диск.  
6. Підключити ISO-образ ОС.  
7. Запустити VM та встановити ОС.  

---

### 2. Апаратні обмеження для 32- та 64-бітних ОС

**32-bit ОС:**
- працює на 32- і 64-бітних процесорах  
- максимум 4 ГБ RAM  

**64-bit ОС:**
- потребує 64-бітний процесор  
- обов’язкова підтримка віртуалізації (Intel VT-x / AMD-V)  
- дозволяє використовувати більше 4 ГБ RAM  

---

### 3. Основні етапи встановлення Linux в текстовому режимі
1. Завантаження з ISO-образу  
2. Вибір мови та клавіатури  
3. Налаштування мережі(можна і без цього, але краще підключити для нових версій пакетів) 
4. Розмітка диска  
5. Встановлення системи

---

### 4. Встановлення GNOME та KDE в Linux

**Ubuntu:**
sudo apt update

sudo apt install ubuntu-gnome-desktop

sudo apt install kubuntu-desktop

**Fedora:**

sudo dnf groupinstall "GNOME Desktop"

sudo dnf groupinstall "KDE Plasma Workspaces"

### 5(1). Характеристика графічних інтерфейсів Глядченко 4-й варіант (KDE, Fluxbox):
**KDE Plasma** — це сучасне, потужне та дуже гнучке графічне середовище для Linux.

Основні характеристики - привабливий, сучасний інтерфейс, дуже широкі можливості налаштування, підтримка віджетів, тем, ефектів, зручний для користувачів Windows.

**Fluxbox** — це легке, швидке та мінімалістичне графічне середовище.

Основні характеристики - дуже низьке споживання ресурсів, висока швидкість роботи, мінімалістичний дизайн, орієнтований на досвідчених користувачів, конфігурується через текстові файли

---

### 5(2). Характеристика графічних інтерфейсів Козуб 11-й варіант (Gnome, JWM):

**Gnome** - це сучасне, стабільне та інтуїтивно зрозуміле графічне середовище, що є стандартом для багатьох дистрибутивів.

Основні характеристики: унікальний візуальний стиль "Activities", орієнтація на робочі простори, висока якість графіки, відмінна підтримка сенсорних екранів та жестів, велика екосистема вбудованих додатків.

**JWM (Joe's Window Manager)** - це надзвичайно легкий, швидкий та компактний менеджер вікон для графічної системи X11.

Основні характеристики: мінімальне використання оперативної пам'яті, класичний інтерфейс з панеллю завдань та меню, ідеальний для застарілого обладнання, конфігурується через редагування XML-файлів.

## Козуб Відповіді  на контрольні запитання

### 1. Порівняйте гіпервізори типу 1 та типу 2, яка між ними відмінність та сфера їх застосування?

 - Тип 1: Працює безпосередньо на апаратному забезпеченні комп'ютера. У нього немає «господарської» ОС. Застосовується у серверах, корпоративних дата-центрах (наприклад, VMware ESXi, Xen).
 - Тип 2: Працює всередині ОС як звичайна програма. Застосовується у навчанні, тестуванні ПЗ, роботі з ОС на домашньому ПК (наприклад, VirtualBox, VMware Workstation)

Головна відмінність: Гіпервізор 1 типу продуктивніший, бо не має зайвого прошарку у вигляді основної ОС

---

### 2. Розкрийте поняття «GNU GPL», яка його основна концепція?
GNU GPL (General Public License) — це ліцензія на вільне програмне забезпечення.
Основна концепція: "Copyleft". Вона гарантує користувачам право запускати, поширювати та змінювати програму. Головна умова: будь-які зміни також мають залишатися відкритими під цією ж ліцензією

---

### 3. В чому суть програмного забезпечення з відкритим кодом (Open Source)?
Суть полягає у відкритості вихідного коду. Будь-яка людина може переглянути, як працює програма, виправити в ній помилки або додати новий функціонал. Це сприяє швидкому розвитку ПЗ зусиллями спільноти та підвищує безпеку, оскільки код прозорий

--- 

### 4. Що таке дистрибутив?
Дистрибутив — це повний набір програмного забезпечення, що включає ядро Linux, системні утиліти, графічне середовище, інсталятор та пакетний менеджер для керування програмами (наприклад, Ubuntu, Debian, Fedora)

---

### 5. Які задачі системного адміністрування можна реалізувати на базі ОС Linux?

  1. Керування користувачами та правами доступу

  2. Налаштування мережевих сервісів

  3. Автоматизація завдань за допомогою скриптів

  3. Моніторинг стану системи

  4. Керування файловими системами та сховищами даних

---

### 6. Як пов'язані між собою ОС Android та Linux?
Android використовує ядро Linux для взаємодії з апаратним забезпеченням телефона. Проте Android не є типовим дистрибутивом Linux, оскільки використовує власну графічну систему

---

### 7. Основні можливості та сфера використання Embedded Linux?
Embedded Linux — це спеціалізовані версії Linux для вбудованих систем

 - Сфера використання: Розумні будинки (IoT), роутери, автомобільні системи, медичне обладнання, телевізори
   
 - Можливості: Компактність, робота на обмеженому залізі, можливість глибокого налаштування під конкретне завдання

---

### 8. Яким чином можна змінити тип завантаження Linux: в текстовому режимі (3 рівень) або графічному (рівень 5)? Чим відрізняються режими CLI та GUI?

1. Зміна режиму: У системах з systemd це робиться командами:

    - sudo systemctl set-default multi-user.target (текстовий режим).
   
    - sudo systemctl set-default graphical.target (графічний режим).

3. Різниця:

    - CLI: Текстовий інтерфейс, де керування йде через введення команд. Швидший, потужніший для адміністрування, споживає мінімум ресурсів.

    - GUI : Графічний інтерфейс з вікнами та іконками. Зручніший для звичайних користувачів, але потребує більше ресурсів.
