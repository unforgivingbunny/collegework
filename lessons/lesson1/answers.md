# Самостійна робота студента

## 1. Контрольні запитання

### 1.1. Що таке дизайн у широкому сенсі?

Дизайн у широкому сенсі — це створення зовнішнього вигляду для систем або середовищ з урахуванням їхньої функціональності та естетики.

### 1.2. Різниця між UI та UX на прикладі ліфта

**UI (User Interface)** — це те, що користувач бачить і з чим взаємодіє: кнопки поверхів, шрифт цифр, підсвітка кнопок, розташування панелі.

**UX (User Experience)** — це загальний досвід користування: наскільки швидко реагує ліфт, чи логічно розташовані кнопки, чи зрозуміло, куди він їде, чи не створює користування стрес.

### 1.3. Обмеження сенсорного екрана смартфона порівняно з монітором ПК

* менший розмір екрана;
* погана точність (немає повнорозмірної клавіатури, на сенсорній великий ризик помилки);
* необхідність більших інтерактивних елементів(якщо інтерфейс застосунку чи сайту маленький то треба вдивлюватись);

### 1.4. Що таке «афорданс» в інтерфейсі?

**Афорданс** — це властивість елемента, яка підказує користувачу, як з ним взаємодіяти.

### 1.5. Класифікація інтерфейсів

Інтерфейси можна класифікувати:

* за типом взаємодії: графічні, голосові, жестові;
* за платформою: мобільні, десктопні, веб;
* за складністю: прості (ATM), складні (професійні CAD-системи);
* за цільовою аудиторією: масові, спеціалізовані.

## 2. Питання на дослідження

### 2.1. 10 принципів хорошого дизайну за Дітером Рамсом

1. Хороший дизайн повинен бути інноваційним.
2. Хороший дизайн робить продукт корисним(застосунок банку).
3. Хороший дизайн повинен бути естетичним(продукти Apple).
4. Хороший дизайн робить продукт зрозумілим(ігри для дітей).
5. Хороший дизайн повинен бути ненав'язливим.
6. Хороший дизайн повинен бути чесним.
7. Хороший дизайн не старіє(годинники Casio).
8. Хороший дизайн ретельно опрацьований до найдрібніших деталей.
9. Хороший дизайн повинен бути чистим (екологічним).
10. Хороший дизайн повинен бути якомога простим.

### 2.2. Accessibility та дизайн в умовах України

Ключові підходи:

* універсальний дизайн: замість окремих пандусів (які часто будують під неправильним кутом), сучасний стандарт — це безбар’єрний вхід на рівні тротуару.
* великі кнопки;
* простір для маневру: збільшення ширини дверних отворів та коридорів у публічних закладах та укриттях.
* прості та зрозумілі формулювання.

### 2.3. Приклади невдалих інтерфейсних рішень

1. Велика кнопка для виходу в конференції в Zoom - дуже легко помилково натиснути.
2. Надмірні сповіщення на сайті aliexpress'у — через це сайт підвисає.

### 2.4. 6 видів UI-інтерфейсів

1. CLI (Command Line Interface)
2. GUI (Graphical User Interface)
3. Voice UI
4. Touch UI
5. Gesture-based UI
6. Web UI
